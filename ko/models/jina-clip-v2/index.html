jina-clip-v2: Multilingual Multimodal Embeddings for Text and Images

개요

Jina CLIP v2는 89개 언어의 시각적 이해와 텍스트 이해 간의 격차를 해소하여 다중 모드 AI에 혁명을 일으켰습니다. 이 모델은 언어 장벽에 관계없이 이미지와 텍스트를 정확하게 일치시켜 글로벌 전자 상거래, 콘텐츠 관리 및 문화 간 커뮤니케이션의 주요 과제를 해결합니다. 국제적으로 사업을 확장하거나 다국어 콘텐츠를 관리하는 기업의 경우 언어마다 별도의 모델이 필요하지 않거나 복잡한 번역 프로세스가 필요하지 않습니다. 이 모델은 글로벌 시장 제품 검색 또는 다국어 디지털 자산 관리와 같이 언어 경계를 넘어 정확한 시각적 검색이 필요한 시나리오에서 특히 효과적입니다.

방법

Jina CLIP v2의 핵심은 Jina XLM-RoBERTa 텍스트 인코더(561M 매개변수)와 EVA02-L14 시각적 인코더(304M 매개변수)를 결합한 복잡한 듀얼 인코더 아키텍처입니다. 텍스트 인코더는 696,320개 토큰의 대규모 컨텍스트 창을 사용하여 89개 언어의 콘텐츠를 처리하는 반면, 시각적 인코더는 최대 512x512픽셀의 고해상도 이미지를 처리합니다. 이 모델은 성능을 유지하면서 1024차원에서 64차원까지 동적 벡터 차원 조정을 가능하게 하는 혁신적인 Matryoshka 표현 학습을 도입합니다. 아키텍처는 각 인코더를 통해 텍스트와 이미지를 처리하여 원래 형식이나 언어에 관계없이 유사한 개념이 정렬될 수 있는 공유 의미 공간에 투영합니다.

성능

이 모델은 Flickr30k 이미지-텍스트 검색 작업에서 98.0%의 정확도를 달성하여 이전 모델과 NLLB-CLIP-SigLIP을 능가하며 최첨단 성능을 달성했습니다. 다국어 시나리오에서 이 모델은 최대 경쟁사보다 매개변수가 적음에도 불구하고 언어 간 이미지 검색 작업에서 NLLB-CLIP-SigLIP보다 4% 향상된 성능을 달성했습니다. 모델은 벡터가 압축된 경우에도 강력한 성능을 유지합니다. 크기를 75% 줄이면 텍스트, 이미지 및 크로스 모달 작업에서 99% 이상의 성능이 유지됩니다. 종합 다국어 MTEB 벤치마크에서는 검색 작업에서 69.86%, 의미 유사성 작업에서 67.77%를 달성해 전문화된 텍스트 벡터 모델과 경쟁할 수 있습니다.

가이드

최적의 배포를 위해 사용자는 몇 가지 주요 요소를 고려해야 합니다. 이 모델에는 효율적인 처리를 위해 CUDA 지원 하드웨어가 필요하며, 배치 크기와 이미지 해상도에 따라 메모리 요구 사항이 조정됩니다. API 비용과 성능을 최적화하려면 처리하기 전에 이미지 크기를 512x512픽셀로 조정하세요. 더 큰 이미지는 자동으로 타일링되어 토큰 사용량과 처리 시간이 늘어납니다. 이 모델은 이미지를 여러 언어의 설명 텍스트와 일치시키는 데 탁월하지만 추상적인 개념이나 고도로 전문화된 도메인별 콘텐츠에는 어려움을 겪을 수 있습니다. 전자상거래 상품 검색, 콘텐츠 추천 시스템, 시각적 검색 애플리케이션에 특히 효과적이지만 세밀한 시각적 세부 분석이나 고도로 전문화된 도메인 전문 지식이 필요한 작업에는 적합하지 않을 수 있습니다. Matryoshka를 사용하여 기능을 표현하는 경우 차원 축소와 성능 간의 균형을 고려하십시오. 64차원 벡터는 강력한 성능을 유지하는 반면 중요한 응용 프로그램은 더 높은 차원의 이점을 누릴 수 있습니다.
