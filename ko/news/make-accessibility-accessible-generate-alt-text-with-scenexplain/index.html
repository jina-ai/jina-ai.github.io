웹 접근성을 더 쉽게: SceneXplain API로 대체 텍스트 만들기

SceneXplain은 시각 장애가 있는 사용자를 돕고 SEO를 개선하기 위해 이미지 alt 텍스트를 쉽게 생성할 수 있게 해주는 접근성 도우미입니다
Alex C-G • 4 독서의 분

접근성(줄여서 "a11y")은 웹 개발과 전자상거래에서 중요한 부분이 되어가고 있습니다. 예전에는 개발자와 기업들이 대체 텍스트나 색맹을 위한 색상 구성과 같은 접근성 보조 도구를 우선순위로 두지 않았습니다. 하지만 이제 유럽과 미국의 접근성 법률로 인해, 웹사이트를 접근 가능하게 만드는 것이 그 어느 때보다 중요해졌습니다.

💡
대체 텍스트는 웹사이트와 디지털 문서에서 사용되는 이미지에 대한 간단한 설명입니다. 이는 시각적 장애로 인해 스크린 리더를 사용하는 사람들이나 인터넷 속도가 느려 이미지가 로드되지 않는 사람들이 이미지의 내용을 이해하는 데 도움을 줍니다. 대체 텍스트는 검색 엔진이 이미지를 이해하고 인덱싱하는 데도 유용합니다.

하지만 웹사이트의 모든 이미지에 대체 텍스트를 어떻게 생성할 수 있을까요? 각 이미지를 수동으로 확인하고 대체 텍스트를 작성하는 것은 특히 수천(또는 수백만) 개의 이미지가 있다면 영원히 걸릴 수 있습니다. 그리고 매일 새로운 이미지가 추가된다면, 이는 끝나지 않는 싸움이 됩니다.

여기서 SceneXplain이 등장합니다. 이것은 당신의 접근성 동맹입니다! 단순히 이미지를 업로드하면 직접 문구를 고민할 필요 없이 대체 텍스트를 얻을 수 있습니다.

수십 개의 이미지가 있다면, 이는 당신의 두뇌에 휴식을 주는 좋은 방법입니다. 하지만 여전히 모든 클릭과 드래그를 직접 해야 합니다. 두뇌는 이득이지만, 손가락은 그렇지 않죠. 그리고 수천 개의 이미지가 있다면? 지금 바로 수근관 증후군 진료 예약을 하세요.

전체 과정을 자동화할 수 있는 방법이 있다면 좋겠죠. 그러면 당신의 두뇌와 손가락 모두 더 흥미로운 일에 집중할 수 있을 것입니다.

여기서 SceneXplain의 API가 등장합니다. 수천 개의 이미지를 처리하고, 이를 배치로 SceneXplain에 전송하여 결과를 CSV 파일로 생성하는 스크립트를 작성할 수 있습니다(또는 약간의 추가 코딩으로 워크플로우에 직접 통합할 수 있습니다).

결국, 사람들이 말하듯이. 행복(happiness)에는 API가 있어야 합니다(APIness).

💡
다시 생각해보니, 영어는 사실 그렇게 작동하지 않는다는 것을 깨달았습니다.

간단히 말해서, Python(또는 다른 언어)으로 프로그램을 작성하여 SceneXplain의 API를 통해 전체 대체 태그 작성 과정을 자동화할 수 있습니다. 우리는 이를 수행하는 Python 코드 스니펫을 가지고 있습니다.

💡
자신의 컴퓨터에서 코드를 실행할 수 없나요? 브라우저에서 사용할 수 있는 Google Colab 노트북을 계속 읽어보세요.

사용 방법은 다음과 같습니다:

  1. requests 라이브러리 설치:
pip install requests
  1. SceneXplain의 API 페이지에서 시크릿 키를 생성하고 클립보드에 복사합니다.
  2. 아래 Python 코드에 붙여넣습니다.
  3. 이미지 URL을 ....이라고 표시된 곳에 복사합니다.
  4. 코드를 실행하세요!
import requests
import json

# generate token on SceneXplain's API page
YOUR_GENERATED_SECRET = "your_generated_secret_here"

data = {
  "data": [
    {
      "task_id": "alt_text",
      "languages": [
        "en"
      ],
      "image": "..." # change to image URL
    }
  ]
}

headers = {
  "x-api-key": f"token {YOUR_GENERATED_SECRET}",
  "content-type": "application/json",
}

response = requests.post("https://api.scenex.jina.ai/v1/describe", headers=headers, json=data)
print(response.json())

(나중에 cURL과 JavaScript용 코드 스니펫도 추가하겠습니다)

tag실제 작동하는 API: 노트북의 SceneXplain

이것을 실제로 보기 위해, 노트북에서 코드를 실시간으로 사용해보겠습니다. 이를 통해 실제 데이터로 무슨 일이 일어나는지 실시간으로 볼 수 있고, Python 코드를 직접 살펴보고 실험해볼 수 있습니다.

이 노트북은 위의 간단한 Python 스니펫을 넘어서 샘플 데이터셋을 다운로드하고 결과를 CSV 파일로 내보냅니다.

tag노트북을 넘어서: 실제 환경에서 API 사용하기

물론 SceneXplain의 API를 사용할 때 Python에만 국한되지 않습니다. HTTP 라이브러리가 있는 모든 언어에서 잘 작동할 것입니다.

위의 동일한 코드 스니펫을 JavaScript로 작성하면 다음과 같습니다:

const body = {
  "data": [
    {
      "task_id": "alt_text",
      "languages": [
        "en"
      ],
      "image": "..."
    }
  ]
};

const YOUR_GENERATED_SECRET = 'your_generated_secret_here';

fetch('https://api.scenex.jina.ai/v1/describe', {
  headers: {
    'x-api-key': `token ${YOUR_GENERATED_SECRET}`,
    'content-type': 'application/json'
  },
  body: JSON.stringify(body),
  method: 'POST'
}).then(async (resp) => {
  if (resp.ok) {
    const data = await resp.json();
    console.log(data);
  }
});

그리고 cURL 명령어로는 다음과 같습니다:

curl "https://api.scenex.jina.ai/v1/describe" \
  -H "x-api-key: token $YOUR_GENERATED_SECRET" \
  -H "content-type: application/json" \
  --data '{
  "data": [
    {
      "task_id": "alt_text",
      "languages": [
        "en"
      ],
      "image": "..."
    }
  ]
}'
