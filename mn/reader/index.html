Уншигч гэж юу вэ?

Вэб мэдээллийг LLM-д оруулах нь үндэслэлтэй болгох чухал алхам боловч хэцүү байж болно. Хамгийн энгийн арга бол вэб хуудсыг хусах, түүхий HTML-г оруулах явдал юм. Гэсэн хэдий ч хусах нь нарийн төвөгтэй бөгөөд ихэвчлэн хаагддаг бөгөөд түүхий HTML нь тэмдэглэгээ, скрипт гэх мэт гадны элементүүдээр дүүрсэн байдаг. Reader API нь URL-аас үндсэн агуулгыг гаргаж аваад цэвэр, LLM-д ээлтэй текст болгон хөрвүүлэх замаар эдгээр асуудлыг шийдэж, таны агент болон RAG системд өндөр чанартай оролтыг баталгаажуулдаг.

Хайлтын үндэслэлд зориулсан уншигч

LLM нь мэдлэгийн хязгаартай тул дэлхийн хамгийн сүүлийн үеийн мэдлэгийг олж авах боломжгүй гэсэн үг юм. Энэ нь ташаа мэдээлэл, хуучирсан хариулт, хий үзэгдэл болон бусад бодит байдлын асуудлууд зэрэг асуудлуудад хүргэдэг. Газардуулга нь GenAI програмуудад зайлшгүй шаардлагатай. Уншигч танд LLM-ээ вэб дэх хамгийн сүүлийн үеийн мэдээллээр суурьлуулах боломжийг олгоно. Асуулгадаа https://s.jina.ai/ гэж бичээд өгвөл Уншигч вэбээс хайгаад шилдэг таван илэрцийг URL болон контентын хамт тус бүр нь цэвэр, LLM-д ээлтэй текстээр харуулна. Ингэснээр та LLM-ээ үргэлж шинэчилж, бодит байдлыг сайжруулж, хий үзэгдэлийг багасгах боломжтой.

Баримт шалгах уншигч

Газардуулгын шинэ төгсгөлийн цэг нь бодит цагийн ойролцоо үнэнийг шалгах туршлагыг санал болгодог. Энэ нь өгөгдсөн мэдэгдлийг авч, бодит цагийн вэб хайлтын үр дүнг ашиглан үндэслэлээ гаргаж, бодит байдлын оноо, ашигласан лавлагаа зэргийг буцаана. Та LLM хий үзэгдэлийг багасгах эсвэл хүний ​​бичсэн контентын бүрэн бүтэн байдлыг сайжруулахын тулд мэдэгдлийг хялбархан үндэслэл болгож чадна.

Уншигч бас зургуудыг уншдаг!

Вэб хуудсан дээрх зургуудыг уншигчийн харааны хэлний загварыг ашиглан автоматаар тайлбар хийж, гаралт дахь зургийн alt шошго хэлбэрээр форматлана. Энэ нь таны доод түвшний LLM-д эдгээр зургуудыг үндэслэл, нэгтгэн дүгнэх үйл явцдаа оруулах хангалттай зөвлөмжийг өгдөг. Энэ нь та зургуудын талаар асуулт асуух, тодорхой нэгийг нь сонгох, эсвэл илүү гүнзгий дүн шинжилгээ хийхийн тулд URL-уудыг илүү хүчирхэг VLM рүү дамжуулах боломжтой гэсэн үг юм!

Уншигч PDF файлуудыг бас уншдаг!

Тийм ээ, Reader нь PDF уншихыг дэмждэг. Энэ нь олон зурагтай зэрэг ихэнх PDF файлуудтай нийцдэг бөгөөд маш хурдан юм! LLM-тэй хослуулснаар та ChatPDF эсвэл баримт бичгийн шинжилгээний хиймэл оюун ухааныг богино хугацаанд хялбархан бүтээх боломжтой.

API үнэ

Манай API үнэ нь хүсэлтэд илгээсэн жетонуудын тоо хэмжээгээр тодорхойлогддог. Reader API-ийн хувьд энэ нь хариултууд дахь жетонуудын хэмжээ юм. Энэхүү үнийн загвар нь Jina AI-ийн хайлтын сангийн бүх бүтээгдэхүүнд хамаарна: Embedding, Reranking, Reader, Auto Fine-Tuning API. Ижил API түлхүүрээр та бүх API үйлчилгээнд хандах боломжтой.
Jina Search Foundation API-тай
Манай бүх бүтээгдэхүүнд хандах хамгийн хялбар арга. Явж байхдаа токенуудыг цэнэглэ.
Токен бага байх үед автоматаар цэнэглэ
Үйлдвэрлэлд тасралтгүй үйлчилгээ үзүүлэхийг зөвлөж байна. Таны токены үлдэгдэл таны тогтоосон босго хэмжээнээс доогуур байвал бид таны кредит картыг хамгийн сүүлд цэнэглэсэнтэй ижил хэмжээгээр автоматаар цэнэглэх болно. Хэрэв та сүүлийн цэнэглэлтээр олон багц худалдаж авсан бол бид зөвхөн нэг багцыг цэнэглэх болно.

Reader API ашиглахтай холбоотой ямар зардал гарах вэ?
Reader API нь үнэ төлбөргүй бөгөөд API түлхүүр шаарддаггүй. Зүгээр л өөрийн URL дээр 'https://r.jina.ai/' гэж бичнэ үү.
Reader API хэрхэн ажилладаг вэ?
Reader API нь прокси ашиглан дурын URL-г дуудаж, агуулгыг нь вэб хөтөч дээр буулгаж, өндөр чанартай үндсэн контентыг гаргаж авдаг.
Reader API нээлттэй эх сурвалж мөн үү?
Тийм ээ, Reader API нь нээлттэй эх сурвалж бөгөөд Jina AI GitHub репозитор дээр байдаг.
Reader API-ийн ердийн хоцролт гэж юу вэ?
Reader API нь ерөнхийдөө URL-уудыг боловсруулж, 2 секундын дотор агуулгыг буцаана, гэхдээ нарийн төвөгтэй эсвэл динамик хуудсууд илүү их цаг шаардаж магадгүй юм.
Би яагаад хуудсыг өөрөө хусахын оронд Reader API ашиглах ёстой вэ?
Ялангуяа нарийн төвөгтэй эсвэл динамик хуудсуудыг хусах нь төвөгтэй бөгөөд найдваргүй байж болно. Reader API нь цэвэр, LLM-д бэлэн текстийн оновчтой, найдвартай гаралтыг өгдөг.
Reader API нь олон хэлийг дэмждэг үү?
Reader API нь контентыг URL-ийн эх хэлээр буцаана. Энэ нь орчуулгын үйлчилгээ үзүүлдэггүй.
Хэрэв вэбсайт Reader API-г хаасан бол би яах ёстой вэ?
Хэрэв танд блоклох асуудал тулгарвал манай тусламжийн багтай холбогдож тусламж, шийдвэрлэнэ үү.
Reader API нь PDF файлаас контент гаргаж чадах уу?
Үндсэндээ вэб хуудсуудад зориулагдсан боловч Reader API нь arXiv гэх мэт вэб сайтууд дээр HTML форматаар үзсэн PDF-ээс контент гаргаж авах боломжтой боловч ерөнхий PDF задлахад оновчтой биш юм.
Reader API нь вэб хуудаснаас медиа контентыг боловсруулж чадах уу?
Одоогоор Reader API нь медиа контентыг боловсруулдаггүй ч ирээдүйн сайжруулалтад зургийн тайлбар, видео хураангуй орно.
Орон нутгийн HTML файлууд дээр Reader API ашиглах боломжтой юу?
Үгүй ээ, Reader API нь зөвхөн нийтэд нээлттэй URL-уудын контентыг боловсруулах боломжтой.
Reader API контентыг кэш хийдэг үү?
Хэрэв та ижил URL хаягийг 5 минутын дотор хүсэх юм бол Reader API нь кэштэй контентыг буцаана.
Би нэвтрэлтийн цаана байгаа контентод хандахын тулд Reader API ашиглаж болох уу?
Харамсалтай нь үгүй.
Би arXiv дээр PDF руу хандахын тулд Reader API ашиглаж болох уу?
Тийм ээ, та Уншигчаас (https://r.jina.ai/https://arxiv.org/pdf/2310.19923v4) уугуул PDF дэмжлэгийг ашиглах эсвэл arXiv (https://)-ийн HTML хувилбарыг ашиглаж болно. r.jina.ai/https://arxiv.org/html/2310.19923v4)
Reader дээр зургийн тайлбар хэрхэн ажилладаг вэ?
Уншигч заасан URL дээрх бүх зургийн тайлбарыг бичээд `Image [idx]: [ caption]`-г alt шошго болгон нэмнэ (хэрэв тэдгээрт анх байхгүй бол). Энэ нь доод түвшний LLM-д тайлбар хийх, нэгтгэн дүгнэх гэх мэт зургуудтай харилцах боломжийг олгодог.
Уншигчийн өргөтгөх чадвар юу вэ? Би үүнийг үйлдвэрлэлд ашиглаж болох уу?
Reader API нь маш томруулж болохуйцаар бүтээгдсэн. Энэ нь бодит цагийн траффик дээр тулгуурлан автоматаар хэмжигддэг бөгөөд хамгийн дээд зэрэглэлийн хүсэлт нь одоо 4000 орчим байна. Бид үүнийг Jina AI-ийн үндсэн бүтээгдэхүүний нэг болгон идэвхтэй хадгалж байна. Тиймээс үйлдвэрлэлд чөлөөтэй ашиглах боломжтой.
Reader API-ийн хурдны хязгаар хэд вэ?
Доорх хүснэгтээс ханшийн хязгаарын хамгийн сүүлийн үеийн мэдээллийг авна уу. Бид Reader API-ийн хурдны хязгаар болон гүйцэтгэлийг сайжруулахаар идэвхтэй ажиллаж байгаа тул хүснэгтийг зохих ёсоор шинэчлэх болно гэдгийг анхаарна уу.
