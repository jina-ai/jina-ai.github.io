Ангилагч

Мультимодал болон олон хэлтэй өгөгдөлд зориулсан өндөр гүйцэтгэлтэй тэг ба цөөн цохилтын ангилагч.

Ангилагч гэж юу вэ?

Ангилагч нь оруулах загвар (jina-embeddings-v3 болон jina-clip-v1) ашиглан текст болон зургийг ангилдаг API үйлчилгээ бөгөөд сургалтгүйгээр 0-shot ангиллыг хоёуланг нь дэмждэг. өгөгдөл болон хамгийн бага жишээ бүхий цөөн тооны сургалт.
Хувилбарын тэмдэглэлийг уншина ууarrow_forward

API үнэ

Манай API үнэ нь хүсэлтэд илгээсэн жетонуудын тоо хэмжээгээр тодорхойлогддог. Reader API-ийн хувьд энэ нь хариултууд дахь жетонуудын хэмжээ юм. Энэхүү үнийн загвар нь Jina AI-ийн хайлтын сангийн бүх бүтээгдэхүүнд хамаарна: Embedding, Reranking, Reader, Auto Fine-Tuning API. Ижил API түлхүүрээр та бүх API үйлчилгээнд хандах боломжтой.
Jina Search Foundation API-тай
Манай бүх бүтээгдэхүүнд хандах хамгийн хялбар арга. Явж байхдаа токенуудыг цэнэглэ.
Токен бага байх үед автоматаар цэнэглэ
Үйлдвэрлэлд тасралтгүй үйлчилгээ үзүүлэхийг зөвлөж байна. Таны токены үлдэгдэл таны тогтоосон босго хэмжээнээс доогуур байвал бид таны кредит картыг хамгийн сүүлд цэнэглэсэнтэй ижил хэмжээгээр автоматаар цэнэглэх болно. Хэрэв та сүүлийн цэнэглэлтээр олон багц худалдаж авсан бол бид зөвхөн нэг багцыг цэнэглэх болно.

Бусад асуултууд

Ангилагчтай холбоотой нийтлэг асуултууд
Цөөн шидэлт болон тэг цохилтын шошго нь юугаараа ялгаатай вэ?
Zero-shot нь ангиллын үед семантик шошго шаарддаг бөгөөд сургалтын явцад байхгүй бол цөөн тооны цохилт нь сургалтын явцад шошго шаарддаг боловч ангилдаггүй. Энэ нь 0-shot нь уян хатан, яаралтай ангиллын хэрэгцээнд илүү тохиромжтой, харин цөөн шидэлт нь тогтмол, тодорхой домэйны категориудад илүү сайн бөгөөд цаг хугацааны явцад өөрчлөгддөг гэсэн үг юм.
Num_iters гэж юу вэ, үүнийг хэрхэн ашиглах вэ?
num_iters нь сургалтын эрчмийг хянадаг - өндөр утгууд нь чухал жишээнүүдийг бататгадаг бол бага утга нь найдвартай бус мэдээллийн нөлөөллийг бууруулдаг. Үүнийг сүүлийн үеийн жишээнүүдэд илүү их давталтын тоогоор өгснөөр цаг хугацааны мэдлэгийг хэрэгжүүлэхэд ашиглаж болох бөгөөд энэ нь өгөгдлийн хэлбэрийг өөрчлөхөд үнэ цэнэтэй болгодог.
Нийтийн ангилагчийг хуваалцах нь хэрхэн ажилладаг вэ?
Нийтийн ангилагчийг classifier_id-тэй хүн бүр өөрийн токен квотыг ашиглан ашиглаж болно. Хэрэглэгчид сургалтын өгөгдөл эсвэл тохиргоонд хандах боломжгүй, бусдын ангиллын хүсэлтийг харах боломжгүй тул ангилагчийг аюулгүйгээр хуваалцах боломжтой.
Сайн ажиллахын тулд цөөн тооны шидэлт хийхэд хэр их мэдээлэл хэрэгтэй вэ?
Цөөн цохилтын ангиллыг давахын тулд 200-400 сургалтын жишээ шаардлагатай. Энэ нь эцсийн дүндээ өндөр нарийвчлалтай болох ч үр дүнтэй болохын тулд энэ халаалт шаардлагатай. Zero-shot нь сургалтын өгөгдөлгүйгээр шууд тогтвортой гүйцэтгэлийг хангадаг.
Энэ нь олон хэл болон текст/зураг хоёуланг нь зохицуулж чадах уу?
Тийм - API нь jina-embeddings-v3 болон URL эсвэл base64 кодлогдсон зургуудыг дэмждэг jina-clip-v1-г ашиглан мультимодаль (текст/зураг) ангиллыг ашиглан олон хэлний асуулга дэмждэг. ижил хүсэлтээр.
Миний мэдэх ёстой хатуу хязгаарлалтууд юу вэ?
Zero-shot нь ангилагчийн хязгаарлалтгүйгээр 256 ангиллыг дэмждэг бол цөөн тооны цохилт нь 16 анги, 16 ангилагчаар хязгаарлагддаг. Аль аль нь хүсэлт бүрт 1024 оролт, нэг оролт бүрт 8192 жетон дэмждэг.
Цаг хугацааны явцад өгөгдлийн өөрчлөлтийг хэрхэн зохицуулах вэ?
Цөөн цохилтын горим нь өөрчлөгдөж буй өгөгдлийн загварт дасан зохицохын тулд /train төгсгөлийн цэгээр дамжуулан тасралтгүй шинэчлэх боломжийг олгодог. Ангилагчийг бүхэлд нь дахин бүтээхгүйгээр өгөгдлийн хуваарилалт өөрчлөгдөхөд та шинэ жишээ эсвэл ангиудыг аажмаар нэмж болно.
Би илгээсний дараа миний сургалтын өгөгдөлд юу тохиолдох вэ?
API нь нэг удаагийн онлайн сургалтыг ашигладаг - сургалтын жишээнүүд ангилагчийн жинг шинэчилдэг боловч дараа нь хадгалагдахгүй. Энэ нь та сургалтын түүхэн өгөгдлийг олж авах боломжгүй гэсэн үг боловч энэ нь нууцлал, нөөцийн үр ашгийг баталгаажуулдаг.
Тэг шидэлт ба цөөн шидэлт - алийг нь хэзээ ашиглах вэ?
Шууд үр дүнд хүрэхийн тулд мөн утгын шошготой уян хатан ангилал хэрэгтэй үед тэг цохилтоор эхлүүлээрэй. Хэрэв танд 200-400 жишээ байгаа, илүү нарийвчлалтай байх шаардлагатай эсвэл домэйнд зориулсан/цаг хугацааны мэдрэмтгий өгөгдөлтэй ажиллах шаардлагатай бол цөөн тооны зураг авалт руу шилжинэ үү.
Би өөр хэл/даалгавруудад өөр өөр загвар ашиглаж болох уу?
Тийм ээ, та текстийн ангиллын хувьд jina-embeddings-v3 (ялангуяа олон хэлний хувьд сайн) болон мультимодал ангиллын хувьд jina-clip-v1-ийн аль нэгийг сонгож болно. jina-clip-v2 гэх мэт шинэ загварууд гарах үед API-ээр дамжуулан автоматаар ашиглах боломжтой болно.
